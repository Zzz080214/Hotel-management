# 悦栖酒店管理系统 — 角色与功能设计

## 登录机制

统一登录入口，用户只需输入账号密码。后端根据数据库中 `role` 字段自动识别身份，前端据此加载完全不同的功能模块。系统管理员、经理和前台员工看到的页面和路径**没有任何重叠**；客户通过微信小程序完成预订与自助服务。

| 账号 | 密码 | 角色 | 首页 |
|---|---|---|---|
| `admin` | `admin123` | 系统管理员 | /users 系统用户 |
| `manager` | `manager123` | 经理 | /dashboard 经营概览 |
| `staff` | `staff123` | 前台员工 | /staff/room-board 房态看板 |

---

## 系统管理员模块（6 页 · 系统与经营管理）

| # | 页面 | 路径 | 功能说明 |
|---|---|---|---|
| 1 | 系统用户 | /users | 管理管理员、经理、前台员工账号；支持新增、编辑、启停、重置密码 |
| 2 | 经营概览 | /dashboard | 今日营收、入住率、订单趋势、公告列表。运营数据卡片 + 房型入住条 + 最新订单表 |
| 3 | 房型管理 | /room-types | 房型配置、价格调整、库存管理。总览卡片 + 筛选器 + 房型网格 + 配置表 + 房间状态板 |
| 4 | 订单管理 | /orders | 全量订单查询筛选、取消订单、支付渠道分布。可取消待入住和在住订单 |
| 5 | 营业报表 | /reports | 今日/本周/本月营收统计、入住率分析、房型收入明细、ADR/RevPAR 指标、报表导出 |
| 6 | 系统公告 | /notices | 发布编辑公告、公告列表管理、通知开关 |

## 经理模块（5 页 · 经营管理）

| # | 页面 | 路径 | 功能说明 |
|---|---|---|---|
| 1 | 经营概览 | /dashboard | 今日营收、入住率、订单趋势、公告列表 |
| 2 | 房型管理 | /room-types | 房型配置、价格调整、库存管理 |
| 3 | 订单管理 | /orders | 全量订单查询筛选、取消订单 |
| 4 | 营业报表 | /reports | 营收与入住率统计、报表导出 |
| 5 | 系统公告 | /notices | 公告发布与维护 |

## 前台员工模块（3 页 · 前台操作）

| # | 页面 | 路径 | 功能说明 |
|---|---|---|---|
| 1 | 房态看板 | /staff/room-board | **核心工作页**。全部房间按房型分组色块展示（绿=空净房、金=已入住、黄=待清扫、灰=维修）。点击房间弹出操作：办理入住、办理退房、标记已清扫。顶部统计条显示各状态房间数 |
| 2 | 入住登记 | /staff/checkin | 住客信息录入（姓名、手机、证件）、房型选择自动分配可用房号、日期选择、费用预览。右侧"今日预抵客人"列表可一键载入预订信息 |
| 3 | 退房结算 | /staff/checkout | 搜索在住客人（房号/姓名/订单号）、点击展开账单明细、确认退房。右侧"今日预离客人"列表 + 快速统计（在住/预离/已退） |

## 权限隔离

| 层级 | 实现 |
|---|---|
| 路由隔离 | 系统管理员路由 `/users` `/dashboard` `/room-types` `/orders` `/reports` `/notices`，经理路由 `/dashboard` `/room-types` `/orders` `/reports` `/notices`，员工路由 `/staff/room-board` `/staff/checkin` `/staff/checkout`，**路径按角色隔离** |
| 导航守卫 | `router.beforeEach` 检查角色，访客访问对方路径自动跳回自己首页并提示 |
| 菜单隔离 | 侧边栏根据 `role` 渲染完全不同的菜单项和图标，系统管理员比经理多出“系统用户”入口 |
| API 拦截 | 后端 `AuthInterceptor` 拦截所有 `/api/admin/**` 请求，无有效 token 返回 401；系统用户接口仅系统管理员可访问 |

## 登录流程

```
用户输入账号密码 → POST /api/auth/login
  ↓ 失败 → 401 账号或密码错误
  ↓ 成功 → { token, username, role, displayName }
  ↓ role=ADMIN → 挂载 6 个系统管理员路由 → 跳转 /users
  ↓ role=MANAGER → 挂载 5 个经营管理路由 → 跳转 /dashboard
  ↓ role=STAFF → 挂载 3 个前台路由 → 跳转 /staff/room-board
退出 → 清 token → 清状态 → /login
过期 → 任一 API 返回 401 → 自动踢回 /login
```
