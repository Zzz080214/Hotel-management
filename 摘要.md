# 基于SpringBoot和Vue的酒店管理系统 —— 摘要

## 摘要

随着旅游业的快速发展和互联网技术的普及，传统酒店管理模式在运营效率、客户体验和数据管理等方面已难以满足现代化经营需求。本文设计并实现了一套基于SpringBoot和Vue的酒店管理系统——悦栖酒店管理系统，旨在为中小型酒店提供一套从前台接待到后台经营的全流程数字化解决方案。

系统采用前后端分离的B/S架构，后端基于SpringBoot 3.2框架，整合Spring Data JPA实现数据持久化，使用MySQL关系型数据库存储业务数据；前端管理后台采用Vue.js框架构建单页面应用，面向酒店住客的微信小程序采用原生框架开发。系统通过RESTful API实现前后端数据交互，并基于JWT令牌实现身份认证与角色权限控制。

在功能设计上，系统围绕酒店日常运营场景划分了三类用户角色。管理员端涵盖经营概览、房型管理、订单管理、营业报表和系统公告五大模块，支持房型配置与库存调整、订单全生命周期管理、营收统计与ADR/RevPAR等关键指标分析以及报表导出功能。前台员工端聚焦日常操作场景，提供房态看板、入住登记和退房结算三个核心模块，以可视化色块直观展示房间状态（空净房、已入住、待清扫、维修），支持一键办理入住与退房操作。住客端通过微信小程序提供房型浏览与推荐、在线预订与订单查询、公告查看和个人中心等功能，降低了住客的操作门槛。

系统在实现过程中注重工程实践，采用了统一响应格式封装API返回值、全局CORS跨域配置、前端请求层自动回退本地Mock数据等容错机制，确保了系统的健壮性和可维护性。经功能测试与性能测试验证，系统各模块运行稳定，接口响应时间在正常负载下保持在200毫秒以内，能够满足中小型酒店日常运营管理的实际需求。

**关键词：** 酒店管理系统；SpringBoot；Vue.js；微信小程序；前后端分离

## Abstract

With the rapid development of the tourism industry and the widespread adoption of Internet technologies, traditional hotel management approaches have become increasingly inadequate in terms of operational efficiency, customer experience, and data management. This thesis designs and implements a hotel management system based on SpringBoot and Vue — the Yueqi Hotel Management System — aimed at providing small and medium-sized hotels with a comprehensive digital solution covering front-desk operations to back-office management.

The system adopts a front-end and back-end separation architecture. The back-end is built on the SpringBoot 3.2 framework, integrating Spring Data JPA for data persistence and using MySQL as the relational database. The front-end management dashboard is developed as a single-page application using Vue.js, while the guest-facing WeChat Mini Program is built with the native mini-program framework. Data exchange between the front-end and back-end is facilitated through RESTful APIs, with JWT-based authentication and role-based access control.

Functionally, the system defines three user roles aligned with daily hotel operation scenarios. The administrator portal encompasses five modules: business overview, room type management, order management, revenue reports, and system announcements. It supports room configuration and inventory adjustment, full-lifecycle order management, revenue statistics with key metrics such as ADR and RevPAR, and report export. The front-desk staff portal focuses on operational workflows with three core modules: a room status board, check-in registration, and check-out settlement. A visual color-coded board displays room statuses (vacant-clean, occupied, pending cleaning, maintenance) and supports one-click check-in and check-out operations. The guest-facing WeChat Mini Program provides room browsing and recommendations, online booking and order tracking, announcement viewing, and a personal center, lowering the barrier for guest interaction.

The system emphasizes sound engineering practices during implementation, incorporating unified API response formatting, global CORS configuration, and automatic fallback to local mock data in the front-end request layer, ensuring system robustness and maintainability. Functional and performance testing verifies that all modules operate stably, with API response times remaining under 200 milliseconds under normal load, meeting the practical demands of daily operations in small and medium-sized hotels.

**Keywords:** Hotel Management System; SpringBoot; Vue.js; WeChat Mini Program; Front-End and Back-End Separation
